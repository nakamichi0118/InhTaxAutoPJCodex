SOROBOCRは相続税申告に必要な書類をAIで自動読み取りし、財産目録作成を効率化するWebアプリケーションです。

## 1. はじめに

### 推奨環境

- **ブラウザ**: Google Chrome / Microsoft Edge 最新版
- **対応ファイル**: PDF形式（画像の場合は600dpi程度でPDF化）

### 画面構成

```
┌──────────────────────────────────────────────────────────┐
│  SOROBOCR - 相続税申告書類読み取りシステム                  │
├──────────────────────────────────────────────────────────┤
│  [不動産] [有価証券] [預貯金] [生命保険] [その他] [通帳・取引履歴] │
├──────────────────────────────────────────────────────────┤
│                                                          │
│   選択したタブの内容がここに表示されます                     │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### タブ一覧

| タブ名 | 状態 | 説明 |
|-------|------|------|
| **不動産** | 利用可能 | 名寄帳・固定資産評価証明書から土地・家屋を読み取り |
| 有価証券 | 準備中 | 株式・投資信託等の評価（今後対応予定） |
| 預貯金 | 準備中 | 預金残高証明書等（今後対応予定） |
| 生命保険 | 準備中 | 生命保険証券（今後対応予定） |
| その他財産 | 準備中 | その他の財産（今後対応予定） |
| **通帳・取引履歴** | 利用可能 | 通帳PDFから取引明細を読み取り |

## 2. 不動産タブ

名寄帳・固定資産評価証明書・課税明細をアップロードして、土地・家屋の情報を自動抽出します。

### 2.1 基本的な使い方

```
┌─────────────────────────────────────────────────────────┐
│ 不動産                                                   │
├─────────────────────────────────────────────────────────┤
│ 解析エンジン: [Gemini 2.5 Flash ▼]                       │
│                                                         │
│ [PDF ファイルを選択]  複数ファイルの一括読み取りに対応       │
│                                                         │
│ ┌─────────────────────────────────────────────────────┐ │
│ │     ここに PDF をドラッグ＆ドロップ                    │ │
│ │     名寄帳・固定資産評価証明書（nayose）として読み取り  │ │
│ └─────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

**手順:**

1. 解析エンジンを選択（不動産は **Gemini 2.5 Flash** がデフォルト）
2. 「PDF ファイルを選択」をクリックまたはドラッグ＆ドロップ
3. 読み取り完了を待つ

### 2.2 読み取り結果

読み取りが完了すると、以下のような表が表示されます:

```
┌────────────────────────────────────────────────────────────────────┐
│ 登記・地図情報を取得                                                │
├───────┬──────────────────────┬────┬─────┬─────┬────┤
│ 種別  │ 所在地               │地図│登記簿│路線価│図面│
├───────┼──────────────────────┼────┼─────┼─────┼────┤
│[土地] │ 大阪市旭区高殿2丁目88│ ☑ │  ☐  │  ☑  │ ☐ │
│[土地] │ 大阪市旭区高殿2丁目125│ ☑ │  ☐  │  ☑  │ ☐ │
│[家屋] │ 大阪市旭区高殿2丁目88│ ☑ │  ☐  │  ―  │ ☐ │
├───────┴──────────────────────┴────┴─────┴─────┴────┤
│ [情報取得を実行]                                       │
└────────────────────────────────────────────────────────┘
```

### 2.3 取得オプション

| オプション | 対象 | 説明 |
|-----------|------|------|
| **地図** | 土地・家屋 | 位置特定API。Googleマップリンクを生成 |
| **登記簿** | 土地・家屋 | 登記情報PDFを取得（認証情報が必要） |
| **路線価** | 土地のみ | 路線価図画像を取得 |
| **図面** | 土地→地積測量図、家屋→建物図面 | 各種図面を取得（認証情報が必要） |

### 2.4 チェックボックスの連動

> **重要**: 登記簿・路線価・図面を取得するには、**位置特定（地図）が必須**です。

| 操作 | 動作 |
|------|------|
| 登記簿・路線価・図面にチェック | **地図が自動でON** |
| 地図をOFFにしようとする | 他にチェックがあれば警告表示 |

### 2.5 情報取得結果

```
┌─────────────────────────────────────────────────────────────┐
│ 取得結果                                                    │
├─────────────────────────────────────────────────────────────┤
│ [完了] [土地] 大阪市旭区高殿2丁目88                          │
│        [Googleマップ] [路線価図] [国税庁で確認]              │
│                                                             │
│ [完了] [土地] 大阪市旭区高殿2丁目125                         │
│        [Googleマップ] [大阪の倍率表]  ← 路線価図がない場合   │
└─────────────────────────────────────────────────────────────┘
```

### 2.6 路線価図と倍率表

| JON API結果 | 表示されるリンク |
|-------------|-----------------|
| 路線価図が取得できた | `[路線価図]` ボタン + `[国税庁で確認]` |
| 路線価図が取得できなかった（倍率地域） | `[〇〇の倍率表]` リンク |

## 3. 通帳・取引履歴タブ

預貯金通帳や取引履歴のPDFから取引明細を自動抽出します。

### 3.1 基本的な使い方

```
┌─────────────────────────────────────────────────────────┐
│ 通帳・取引履歴                                           │
├─────────────────────────────────────────────────────────┤
│ 読み取り種別: [取引履歴 ▼] [通帳（預貯金）]              │
│                                                         │
│ 解析エンジン: [Gemini 2.5 Pro ▼]                        │
│                                                         │
│ [PDF ファイルを選択]  複数ファイルの一括読み取りに対応    │
│                                                         │
│ ┌─────────────────────────────────────────────────────┐ │
│ │     ここに PDF をドラッグ＆ドロップ                    │ │
│ └─────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

**手順:**

1. 読み取り種別を選択
   - **取引履歴**: 銀行の取引履歴明細
   - **通帳（預貯金）**: 通帳の見開きページ
2. 解析エンジンを選択（通帳は **Gemini 2.5 Pro** がデフォルト・推奨）
3. PDFをアップロード
4. 読み取り完了後、CSVをダウンロード

### 3.2 読み取り結果

```
┌─────────────────────────────────────────────────────────┐
│ 読み取り結果                                             │
├─────────────────────────────────────────────────────────┤
│ 処理時間: 45.2秒                                         │
│ 取引件数: 127件                                          │
│ 入金合計: ¥3,456,789                                     │
│ 出金合計: ¥2,345,678                                     │
├─────────────────────────────────────────────────────────┤
│ [CSVダウンロード]  [JSONダウンロード]  [出納帳に転記]     │
└─────────────────────────────────────────────────────────┘
```

### 3.3 出力CSVの形式

| 列名 | 説明 |
|------|------|
| 日付 | 取引日（YYYY-MM-DD形式） |
| 摘要 | 取引内容 |
| 出金 | 出金額（円） |
| 入金 | 入金額（円） |
| 残高 | 取引後残高（円） |
| メモ | 補足情報 |

## 4. 解析エンジンの選択

### タブ別デフォルト設定

| タブ | デフォルト | 理由 |
|------|-----------|------|
| 不動産 | **Gemini 2.5 Flash** | 表形式データは高速処理で十分 |
| 通帳・取引履歴 | **Gemini 2.5 Pro** | 金額精度が重要なため高精度モード |

### エンジン比較

| エンジン | 精度 | 速度 | 推奨用途 |
|---------|------|------|---------|
| **Gemini 2.5 Pro** | 約99% | 標準 | 通帳・取引履歴、金額精度が重要な場合 |
| **Gemini 2.5 Flash** | 約95% | 高速 | 不動産、大量処理、確認作業 |

## 5. 結果の活用

### CSVダウンロード

読み取り結果は以下の形式でダウンロードできます:

- **bank_transactions.csv**: 取引履歴（通帳タブ）
- **land.csv**: 土地情報（不動産タブ）
- **building.csv**: 家屋情報（不動産タブ）

### Excel財産目録への転記

1. CSVをダウンロード
2. Excelで開く（UTF-8 BOM形式なので文字化けしません）
3. 財産目録シートにコピー＆ペースト

### 出納帳アプリ連携

通帳・取引履歴の読み取り結果は「出納帳に転記」ボタンで直接連携できます。

## 6. よくある質問

### 全般

**Q. 対応している書類は？**

A. 現在以下に対応しています:
- 名寄帳・固定資産評価証明書・課税明細（土地・家屋）
- 預貯金通帳・取引履歴

**Q. アップロードしたデータは安全ですか？**

A. アップロードされたPDFは処理後すぐ破棄されます。サーバーには保存されません。

### 不動産タブ

**Q. 路線価図が取得できない場合は？**

A. 倍率地域の可能性があります。「〇〇の倍率表」リンクから国税庁サイトで確認してください。

**Q. 登記簿が取得できない**

A. 登記簿取得には認証情報の設定が必要です。管理者に連絡してください。

### 通帳・取引履歴タブ

**Q. 読み取り精度はどのくらい？**

A. Gemini 2.5 Proで約99%です。必ず結果を目視確認してください。

**Q. 銀行によって精度に差はありますか？**

A. ゆうちょ・都市銀行は高精度です。独自フォーマットや潰れた文字は誤認識の可能性があります。

## 7. トラブルシューティング

### 読み取りが遅い・完了しない

1. PDFのページ数・ファイルサイズが大きい可能性
2. ページを再読み込みして再アップロード
3. 大きいPDFは分割してアップロード

### 結果が空・文字化け

1. PDF形式を確認（画像PDFは600dpi推奨）
2. ページ向きを確認（横向きでも自動補正されます）

### JON API関連のエラー

1. **位置特定失敗**: 住所表記を確認・修正してください
2. **認証エラー**: 管理者に環境設定を確認してもらってください
